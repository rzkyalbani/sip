Sistem Informasi Manajemen Perpustakaan

Fitur Umum Sistem Informasi Manajemen Perpustakaan (*{} berarti disisihkan terlebih dahulu)
1. Manajemen Buku
Katalog Buku: Pencatatan detail buku (judul, penulis, penerbit, tahun terbit, ISBN, kategori/genre)
Penambahan Buku Baru: Form untuk menambahkan buku ke inventaris
Edit & Hapus Buku: Kemampuan memperbarui atau menghapus data buku
Barcode/QR Code: Sistem pelabelan untuk identifikasi cepat {}
Pencarian & Filter: Kemampuan mencari buku berdasarkan berbagai kriteria
2. Manajemen Anggota
Pendaftaran Anggota: Form untuk mendaftarkan anggota baru
Profil Anggota: Data anggota seperti nama, kontak, alamat, dll
Kartu Anggota: Fitur untuk mencetak/menampilkan kartu anggota {}
Manajemen Status: Aktif, tidak aktif, diblokir, dll
3. Transaksi Peminjaman
Peminjaman Buku: Proses peminjaman dengan periode waktu tertentu
Pengembalian Buku: Proses pencatatan buku yang dikembalikan
Perpanjangan: Opsi untuk memperpanjang masa pinjam
Riwayat Peminjaman: Catatan semua aktivitas peminjaman per anggota
4. Manajemen Denda
Kalkulasi Denda: Menghitung denda untuk keterlambatan
Pembayaran Denda: Mencatat pembayaran denda
Pengingat: Notifikasi jatuh tempo dan denda
5. Dashboard & Laporan
Statistik: Buku paling populer, statistik peminjaman, dll
Laporan Harian/Mingguan/Bulanan: Ringkasan aktivitas perpustakaan
Laporan Inventaris: Status koleksi buku (tersedia, dipinjam, hilang, rusak)
Laporan Keuangan: Denda yang terkumpul dan transaksi lainnya
6. Fitur Tambahan
Reservasi Buku: Anggota dapat memesan buku yang sedang dipinjam
Manajemen Rak: Pelacakan lokasi fisik buku di perpustakaan
Katalog Online: Anggota dapat mencari buku secara online
Ulasan & Rating: Anggota dapat memberikan ulasan dan rating untuk buku
Pengingat Email/SMS: Notifikasi untuk jatuh tempo, buku tersedia, dll



Struktur Database:
Books: Menyimpan informasi tentang buku
Members: Data anggota perpustakaan
Borrowings: Catatan peminjaman dan pengembalian
Categories: Kategori/genre buku
Authors: Informasi tentang penulis
Publishers: Informasi penerbit
Fines: Catatan denda

Entitas Utama dan Relasinya
1. Books (Buku)
    id: int (PK)
    title: varchar
    author_id: int (FK)
    publisher_id: int (FK)
    isbn: varchar
    publication_year: year
    category_id: int (FK)
    description: text
    quantity: int
    available_quantity: int
    location: varchar (rak/posisi)
    created_at: timestamp
    updated_at: timestamp
2. Authors (Penulis)
    id: int (PK)
    name: varchar
    bio: text (opsional)
    created_at: timestamp
    updated_at: timestamp
3. Publishers (Penerbit)
    id: int (PK)
    name: varchar
    address: text (opsional)
    contact: varchar (opsional)
    created_at: timestamp
    updated_at: timestamp
4. Categories (Kategori)
    id: int (PK)
    name: varchar
    description: text (opsional)
    created_at: timestamp
    updated_at: timestamp
5. Members (Anggota)
    id: int (PK)
    member_code: varchar (kode unik anggota)
    name: varchar
    email: varchar
    phone: varchar
    address: text
    status: enum ('active', 'inactive', 'blocked')
    join_date: date
    created_at: timestamp
    updated_at: timestamp
6. Borrowings (Peminjaman)
    id: int (PK)
    member_id: int (FK)
    book_id: int (FK)
    borrow_date: date
    due_date: date
    return_date: date (null jika belum dikembalikan)
    status: enum ('borrowed', 'returned', 'overdue')
    created_by: int (FK ke Users - Admin yang menginput)
    created_at: timestamp
    updated_at: timestamp
7. Fines (Denda)
    id: int (PK)
    borrowing_id: int (FK)
    amount: decimal
    paid_amount: decimal
    paid_date: date (null jika belum dibayar)
    status: enum ('unpaid', 'partially_paid', 'paid')
    created_at: timestamp
    updated_at: timestamp
8. Users (Pengguna/Admin)
    id: int (PK)
    name: varchar
    email: varchar
    password: varchar
    role: enum ('admin', 'staff')
    created_at: timestamp
    updated_at: timestamp

Relasi Antar Entitas
    Books - Authors: Many-to-One (Satu penulis dapat menulis banyak buku)
    Books - Publishers: Many-to-One (Satu penerbit dapat menerbitkan banyak buku)
    Books - Categories: Many-to-One (Satu kategori dapat memiliki banyak buku)
    Members - Borrowings: One-to-Many (Satu anggota dapat meminjam banyak buku)
    Books - Borrowings: One-to-Many (Satu buku dapat dipinjam berkali-kali)
    Borrowings - Fines: One-to-One (Satu peminjaman dapat menghasilkan satu denda)
    Users - Borrowings: One-to-Many (Satu admin dapat mencatat banyak transaksi)
